# 拍照识菜生成器 - MVP 需求文档

## 1. 项目概述

### 产品名称
拍照识菜 - AI 食谱生成器

### 产品定位
用户拍照上传食材图片，AI 自动识别食材并生成对应的食谱指南。

### 核心价值
- 简化烹饪：拍照即可获得食谱，无需手动输入食材
- 智能识别：AI 视觉识别食材种类
- 个性化推荐：根据识别出的食材生成对应食谱

|--------|----------|
| P0 | 作为用户，我可以通过拍照或上传图片来识别食材 |
| P0 | 作为用户，我可以看到 AI 识别出的食材列表 |

## 3. 功能需求

### 3.1 食材识别
- **输入**：相机拍照 / 相册选择图片
- **处理**：调用视觉 AI 模型识别图片中的食材
- **输出**：食材列表（支持中英文）
- **约束**：支持 JPG、PNG 格式

### 3.2 食材编辑
- 用户可手动添加、删除、修改识别出的食材
- 食材之间用逗号分隔

### 3.3 食谱生成
- **输入**：食材列表
- **输出**：完整食谱，包含：
  - 菜名
  - 食材清单（含具体用量，如 200g、2个）
  - 调料清单（含具体用量，如盐半勺=3g）
  - 烹饪步骤（编号列表）
- **约束**：调料不允许出现"适量"，必须具体量化

### 3.4 中英文切换
- 支持中文/英文界面切换
- 食谱内容同步翻译

---

## 4. 非功能需求

### 4.1 性能
- 图片上传响应时间 < 2秒
- 食材识别响应时间 < 5秒
- 食谱生成响应时间 < 10秒

### 4.2 兼容性
- 支持主流移动端浏览器（Chrome、Safari）
- 响应式布局，适配手机/平板

### 4.3 可用性
- 流程简洁，3步完成：拍照 → 识别 → 生成
- 错误提示清晰，引导用户操作

## 5. 技术方案

### 5.1 技术栈
| 层级 | 技术选型 |
|------|----------|
| 前端 | HTML + CSS + JavaScript（单页应用） |
| API | 硅基流动 (SiliconFlow) |
| 视觉模型 | Qwen/Qwen2-VL-72B-Instruct |
| 文本模型 | Qwen/Qwen2.5-72B-Instruct |

### 5.2 架构图
```
[用户] → [上传图片] → [前端] → [视觉模型API] → [识别食材]
                                    ↓
[前端] ← [生成食谱] ← [文本模型API] ← [食材列表]
```

### 5.3 数据流
1. 用户拍照 → Base64 编码 → 发送给视觉模型
2. 视觉模型返回食材 JSON
3. 用户编辑食材（可选）
4. 发送食材到文本模型
5. 文本模型返回格式化食谱

---

## 6. UI/UX 需求

### 6.1 页面结构
1. **首页**：API Key 设置入口
2. **上传页**：拍照/上传按钮 + 图片预览
3. **识别结果页**：食材标签层叠展示 + 可编辑文本框
4. **食谱页**：菜名 + 食材 + 调料 + 步骤（支持语言切换）

### 6.2 视觉规范
- 主题色：紫色渐变 (#667eea → #764ba2)
- 卡片圆角：20px
- 按钮：渐变背景 + 圆角
- 动画：食材层叠浮入效果

### 6.3 交互流程
```
设置API Key → 拍照/上传 → 识别中(loading) → 显示食材 → 
生成食谱 → 显示食谱 → 切换语言
```

## 8. 已知限制

1. **识别准确性**：依赖于 AI 模型能力，部分复杂菜品可能识别不全
2. **用量精度**：AI 估算的用量为参考值，实际操作需根据口味调整
3. **离线不可用**：需要网络连接调用 API
